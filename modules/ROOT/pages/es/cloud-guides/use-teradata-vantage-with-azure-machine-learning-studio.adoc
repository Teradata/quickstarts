== Descripción general

https://docs.microsoft.com/en-us/azure/machine-learning/studio/what-is-ml-studio[Azure Machine Learning (ML) Studio] es una herramienta colaborativa de arrastrar y soltar que puede usar para crear, probar e implementar soluciones de análisis predictivo en sus datos. ML Studio puede consumir datos de Azure Blob Storage. Esta guía de introducción le mostrará cómo copiar conjuntos de datos de Teradata Vantage a un Blob Storage utilizando la función Jupter Notebook "integrada" de ML Studio. Luego, ML Studio puede utilizar los datos para crear y entrenar modelos de aprendizaje automático e implementarlos en un entorno de producción.

== Procedimiento

=== Configuración inicial


* Durante la creación del espacio de trabajo de ML Studio, es posible que deba crear una "nueva" cuenta de almacenamiento a menos que tenga una en las ubicaciones de disponibilidad actual y elija *Estándar DEVTEST* para *Plan de servicios web* para esta guía de introducción. Inicie sesión en https://portal.azure.com/[Azure Portal], abra su cuenta de almacenamiento y cree un *contenedor* si aún no existe una.
+
image:cloud-guides/{page-image-directory}/image3.png[imagen,width=520,height=126]

* Copie su *nombre de la cuenta de almacenamiento* y *llave* en el bloc de notas que usaremos para que Python3 Notebook acceda a su cuenta de Azure Blob Storage.
+
image:cloud-guides/{page-image-directory}/image4.png[imagen,width=448,height=182]

* Finalmente, abra la propiedad *Configuración* y establezca *_'Se requiere transferencia segura'_* en *_Desactivado_* para permitir que el módulo de importación de datos de ML Studio acceda a la cuenta de almacenamiento de blobs.
+
image:cloud-guides/{page-image-directory}/image5.png[imagen,width=449,height=78]

=== Cargar datos

Para llevar los datos a ML Studio, primero debemos cargar los datos de Teradata Vantage en Azure Blob Storage. Crearemos un ML Jupyter Notebook, instalaremos paquetes de Python para conectarnos a Teradata y guardaremos datos en Azure Blob Storage.

Inicie sesión en https://portal.azure.com/[Azure Portal], vaya a su *espacio de trabajo de ML Studio*, https://studio.azureml.net/[inicie Machine Learning Studio] e *inicie sesión.*

1. Debería ver la siguiente pantalla y hacer clic en *Cuadernos*, asegúrese de estar en la región/espacio de trabajo correcto y haga clic en Notebook *Nuevo*
+
image:cloud-guides/{page-image-directory}/image6.png[imagen,width=519,height=250]

2. Elija *Python3* y *Nombre* a su instancia de cuaderno
+
image:cloud-guides/{page-image-directory}/image7.png[imagen,width=524,height=187]

3. En su instancia de Jupyter Notebook, instale el https://pypi.org/project/teradataml/[paquete Teradata Vantage Python para análisis avanzado]:
+
[source, python]
----
pip install teradataml
----
+
NOTE: No existe validación entre Microsoft Azure ML Studio y el paquete Teradata Vantage Python.

4. Instale la https://pypi.org/project/azure-storage-blob/[biblioteca cliente de Microsoft Azure Storage Blob para Python]:
+
[source, python]
----
!pip install azure-storage-blob
----

5. Importe las siguientes bibliotecas:
+
[source, python]
----
import teradataml as tdml
from teradataml import create_context, get_context, remove_context
from teradataml.dataframe.dataframe import DataFrame
import pandas as pd
from azure.storage.blob import (BlockBlobService)
----

6. Conéctese a Teradata usando el comando:
+
[source, python]
----
create_context(host = '<hostname>', username = '<database user name>', password = '<password>')
----

7. Recupere los datos utilizando el módulo Teradata Python DataFrame:
+
[source, python]
----
train_data = DataFrame.from_table("<table_name>")
----

8. Convierta Teradata DataFrame a Panda DataFrame:
+
[source, python]
----
trainDF = train_data.to_pandas()
----

9. Convierta los datos a CSV:
+
[source, python]
----
trainDF = trainDF.to_csv(head=True,index=False)
----

10. Asigne variables para el nombre de la cuenta, la clave y el nombre del contenedor de Azure Blob Storage:
+
[source, python]
----
accountName="<account_name>"
accountKey="<account_key>"
containerName="mldata"
----

11. Cargue el archivo en Azure Blob Storage:
+
[source, python, id="azure_ml_studio_first_config", role="content-editable emits-gtm-events"]]
----
blobService = BlockBlobService(account_name=accountName, account_key=accountKey)
blobService.create_blob_from_text(containerNAme, 'vTargetMail.csv', trainDF)
----

12. Inicie sesión en Azure Portal, abra la cuenta de Blob Storage para ver el archivo cargado:
+
image:cloud-guides/{page-image-directory}/image17.png[imagen,width=618,height=118]

=== Entrenar el modelo

Usaremos el artículo existente https://docs.microsoft.com/en-us/azure/sql-data-warehouse/sql-data-warehouse-get-started-analyze-with-azure-machine-learning[Analizar datos con Azure Machine Learning] para crear un modelo de aprendizaje automático predictivo basado en datos de Azure Blob Storage. Crearemos una campaña de marketing dirigida para Adventure Works, la tienda de bicicletas, prediciendo si es probable que un cliente compre una bicicleta o no.

==== Importar datos

Los datos están en el archivo de Azure Blob Storage llamado `vTargetMail.csv` que copiamos en la sección anterior.

1. Inicie sesión en https://studio.azureml.net/[Azure Machine Learning Studio] y haga clic en *Experimentos*.
2. Haga clic en *+NUEVO* en la parte inferior izquierda de la pantalla y seleccione *Experimento en blanco*.
3. Introduzca un nombre para su experimento: Marketing dirigido.
4. Arrastre el módulo *Importar datos* debajo de *Entrada y salida de datos* desde el panel de módulos al lienzo.
5. Especifique los detalles de su Azure Blob Storage (nombre de cuenta, clave y nombre de contenedor) en el panel Propiedades.

Ejecute el experimento haciendo clic en *Ejecutar* debajo del lienzo del experimento.

image:cloud-guides/{page-image-directory}/image18.png[imagen,width=401,height=281]

Una vez que el experimento termine de ejecutarse exitosamente, haga clic en el puerto de salida en la parte inferior del módulo Importar datos y seleccione *Visualizar* para ver los datos importados.

image:cloud-guides/{page-image-directory}/image19.png[imagen,width=480,height=359]


==== Limpiar los datos

Para limpiar los datos, elimine algunas columnas que no sean relevantes para el modelo. Para ello:

1. Arrastre el módulo *Seleccionar columnas en el conjunto de datos* debajo de *Transformación de datos < Manipulación* hasta el lienzo. Conecte este módulo al módulo *Importar datos*.
2. Haga clic en *Selector de columna de lanzamiento* en el panel Propiedades para especificar qué columnas desea eliminar.
+
image:cloud-guides/{page-image-directory}/image20.png[imagen,width=456,height=301]
3. Excluya dos columnas: *CustomerAlternateKey* y *GeographyKey*.
+
image:cloud-guides/{page-image-directory}/image21.png[imagen,width=443,height=372]

==== Construir el modelo

Dividiremos los datos 80-20: 80 % para entrenar un modelo de aprendizaje automático y 20 % para probar el modelo. Usaremos los algoritmos de "dos clases" para este problema de clasificación binaria.

1. Arrastre el módulo *Dividir datos* hasta el lienzo y conéctelo con 'Seleccionar columnas en el conjunto de datos'.
2. En el panel de propiedades, introduzca 0,8 para Fracción de filas en el primer conjunto de datos de salida.
+
image:cloud-guides/{page-image-directory}/image22.png[imagen,width=399,height=336]

3. Busque y arrastre el módulo *Árbol de decisión potenciado de dos clases* hasta el lienzo.
4. Busque y arrastre el módulo *Entrenar modelo* hasta el lienzo y especifique las entradas conectándolo a los módulos *Árbol de decisión potenciado de dos clases* (algoritmo ML) y *Dividir* *datos* (datos para entrenar el algoritmo).
+
image:cloud-guides/{page-image-directory}/image23.png[imagen,width=396,height=333]

5. Luego, haga clic en *Selector de columna de lanzamiento* en el panel Propiedades. Seleccione la columna *BikeBuyer* como columna para predecir.
+
image:cloud-guides/{page-image-directory}/image24.png[imagen,width=396,height=330]

==== Puntuar el modelo

Ahora, probaremos cómo se desempeña el modelo con los datos de prueba. Compararemos el algoritmo de nuestra elección con un algoritmo diferente para ver cuál funciona mejor.

1. Arrastre el módulo *Puntuar modelo* hasta el lienzo y conéctelo a los módulos *Entrenar modelo* y *Dividir datos*.
+

image:cloud-guides/{page-image-directory}/image25.png[imagen,width=398,height=334]

2. Busque y arrastre *Máquina de punto Bayes de dos clases* hasta el lienzo del experimento. Compararemos el rendimiento de este algoritmo en comparación con el árbol de decisión potenciado de dos clases.
3. Copie y pegue los módulos Train Model y Score Model en el lienzo.
4. Busque y arrastre el módulo *Evaluar modelo* hasta el lienzo para comparar los dos algoritmos.
5. *Ejecutar* el experimento.
+
image:cloud-guides/{page-image-directory}/image26.png[imagen,width=537,height=351]
6. Haga clic en el puerto de salida en la parte inferior del módulo Evaluate Model y haga clic en Visualizar.
+
image:cloud-guides/{page-image-directory}/image27.png[imagen,width=447,height=199]


Las métricas proporcionadas son la curva ROC, el diagrama de recuperación de precisión y la curva de elevación. Al observar estas métricas, podemos ver que el primer modelo funcionó mejor que el segundo. Para ver lo que predijo el primer modelo, haga clic en el puerto de salida del Score Model y haga clic en Visualizar.

image:cloud-guides/{page-image-directory}/image28.png[imagen,width=464,height=204]

Verá dos columnas más agregadas a su conjunto de datos de prueba.
1. Score Probabilities: la probabilidad de que un cliente sea un comprador de bicicletas.
2. Scored Labels: la clasificación realizada por el modelo - comprador de bicicleta (1) o no (0). Este umbral de probabilidad de etiquetado se establece en 50 % y se puede ajustar.

Comparando la columna BikeBuyer (real) con Scored Labels (predicción), puede ver lo bien que se ha desempeñado el modelo. Como siguientes pasos, puede utilizar este modelo para hacer predicciones para nuevos clientes y publicar este modelo como un servicio web o escribir los resultados en SQL Data Warehouse.
